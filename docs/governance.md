# Governance

> *Planned. PRs welcome.*

How INFINI is governed and how spec decisions are made.

---

## The short version

- The spec is CC-BY-4.0. Anyone can implement it.
- The repo is maintained by NickAiNYC and INFINI contributors.
- Spec changes go through RFCs. See [`CONTRIBUTING.md`](../CONTRIBUTING.md).
- Two-week minimum review on any RFC that touches conformance.
- Major version bumps (`1.x → 2.0`) require a published migration guide. See [`spec/migration.md`](../spec/migration.md).

## The long version

This document will eventually describe:

- The role of maintainers vs. contributors.
- How RFCs are proposed, discussed, and accepted.
- How canonical loops are added and removed.
- How engine adapters enter and leave the compatibility matrix.
- How security disclosures are handled.
- How the registry is governed once it ships.

Until then, the short version is the operating contract.

## Security disclosures

Email security@infini.dev (coming soon) with any security issue. Do not open a public issue for security disclosures.

Until the security email is live, DM a maintainer on GitHub.
