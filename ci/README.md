# INFINI CI

Run your loops on every PR. Catch regressions before merge.

> INFINI CI is a GitHub Action that runs `infini ci` against your Loopfiles. It enforces spec compliance, runs loops against fixtures, and posts a semantic-diff comment when a PR changes a Loopfile.

---

## Quickstart

```yaml
# .github/workflows/infini-ci.yml
name: INFINI CI
on: [pull_request]

jobs:
  infini:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: infini/ci@v1
        with:
          loopfile: ./loops/coding-loop.yaml
          fixtures: ./tests/fixtures/
          expect:   ./tests/expected/
```

---

## What it does

1. **Validates** the Loopfile against [`spec/schema.json`](../spec/schema.json).
2. **Runs** the loop against every fixture in `fixtures/`.
3. **Compares** each run's `run.json` against the expected shape in `expect/`.
4. **Diffs** the Loopfile against the base branch and posts a comment describing the semantic change.
5. **Fails** the PR if validation fails, any run's outcome is not `verified`, or any expected-shape check fails.

---

## Inputs

| Input       | Required | Description |
| ----------- | :------: | ----------- |
| `loopfile`  | yes      | Path to the Loopfile under test. |
| `fixtures`  | yes      | Directory of fixture sets (one subdirectory per fixture). |
| `expect`    | yes      | Directory of expected trace shapes (one per fixture). |
| `engine`    | no       | Engine to run with. Default: `infini` (reference). Set to `hermes` or `openclaw` to test an adapter. |
| `budget`    | no       | Override the Loopfile's `BUDGET` for CI. Useful for keeping CI cheap. |
| `fail_fast` | no       | Stop on first failure. Default: `false`. |

## The diff comment

When a PR changes a Loopfile, INFINI CI posts a comment like:

```text
### INFINI diff — loops/coding-loop.yaml

Objective: unchanged
Agents:    +1 (judge added)
Steps:     +1 (s5 verify_governed added)
Verify:    +1 semantic check (judge:policy_citation>=90)
Budget:    $4 → $6 (minutes: 12 → 18)
Stop when: unchanged

This is a backwards-compatible change. Loops pinned to @1.0 will not
see this change; @1.1 will be published on merge.
```

The comment is generated by `infini diff` and is human-readable, not a raw line diff.

## Engine selection in CI

To test a specific adapter:

```yaml
- uses: infini/ci@v1
  with:
    loopfile: ./adapters/hermes/examples/claim-audit-loop.yaml
    fixtures: ./adapters/hermes/examples/fixtures/
    expect:   ./adapters/hermes/examples/expected/
    engine:   hermes
```

To test hybrid mode:

```yaml
- uses: infini/ci@v1
  with:
    loopfile: ./examples/hybrid-hermes-openclaw/governed-coding-loop.yaml
    fixtures: ./examples/hybrid-hermes-openclaw/fixtures/
    expect:   ./examples/hybrid-hermes-openclaw/expected-output.md
    engine:   hermes+openclaw
```

## Self-hosted runners

For loops that need access to private repos, internal services, or self-hosted models, run INFINI CI on a self-hosted runner:

```yaml
jobs:
  infini:
    runs-on: [self-hosted, infini]
    steps:
      - uses: actions/checkout@v4
      - uses: infini/ci@v1
        with: { ... }
```

The runner needs `infini-cli` installed and your adapter extras (`infini-cli[hermes,openclaw]`).
